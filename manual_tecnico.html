Todas las vistas usan showView(id) que alterna la clase active y hace window.scrollTo(0,0).

3. Flujo de autenticación

Paso 1 — El usuario introduce credenciales y pulsa Entrar. Se guardan en S._pendingUsername / S._pendingPassword. Se cargan las 17 sedes estáticas como fallback y se muestra view-sede.

Paso 2 — El usuario elige sede. Se ejecuta el login real:

POST /auth/login
{ "username": "jose.perez", "password": "...", "interlocutor_id": 2 }

El rol se extrae del JWT si d.role llega null:

const _pay = JSON.parse(atob(d.token.split('.')[1]...));
S.role = _pay.role || _pay.rol || '';
⚠️ El API v6.9 puede devolver role: null en el body. Siempre leer del JWT como fallback.

4. Flujo de registro de albarán (paso 4)

Al confirmar el albarán se ejecutan 4 pasos en secuencia:

// PASO 1: Crear OC
POST /purchasing/orders
{
  "supplier_id": 74,
  "interlocutor_id": 2,
  "reference": "ALB-2026-001",
  "reception_date": "2026-06-27",
  "details": [{
    "supplier_item_name": "PANCETA BACON",
    "item_id": 111,
    "item_type": "sku",
    "unit_of_measure": "g",
    "quantity_requested": 1000,
    "unit_price": 5.50
  }]
}
→ { "id": 32, "order_number": "OC-A3F2B1C4-20260627" }

// PASO 2: Aprobar OC
PUT /purchasing/orders/32/approve
{ "reception_date": "2026-06-27" }

// PASO 3: Crear lote (por ítem)
POST /inventory/batches
{
  "batch_reference": "LOT-20260627-ABC",
  "item_id": 111,
  "item_type": "sku",
  "expiration_date": "2027-06-30",   ← opcional
  "cost_per_unit": 5.50
}
→ { "batch": { "id": 19 } }

// PASO 4: Registrar recepción (por ítem)
POST /inventory/reception
{
  "location_id": 2,
  "batch_id": 19,
  "item_id": 111,
  "item_type": "sku",
  "quantity": 1000,
  "movement_type": "Compra",
  "reference_document": "OC-A3F2B1C4-20260627"
}

7. Variables de entorno (Apache VirtualHost)

SetEnv OMNI_API_BASE     https://api.omni.josepan.app/api/v1
SetEnv ANTHROPIC_API_KEY sk-ant-api03-...
SetEnv OMNI_SERVICE_TOKEN eyJ...   # opcional
⚠️ Nunca incluir claves en archivos del repositorio. Usar siempre SetEnv en el VirtualHost de Apache.

8. Despliegue

# Subir archivos al servidor
rsync -avz --exclude='.git' \
  index.html app.js styles.css .htaccess api/ \
  usuario@servidor:/var/www/omni/omni_compras/

# Verificar permisos
chmod 644 /var/www/omni/omni_compras/api/config.php

# Limpiar caché (si fuera necesario)
touch /var/www/omni/omni_compras/index.html

Para forzar recarga en el navegador, incrementar el parámetro de versión en index.html:

<script src="app.js?v=YYYYMMDD" defer></script>

9. CSS — Tailwind compilado estático

El proyecto usa Tailwind CSS compilado con CLI v3 (no CDN). Para recompilar tras cambios en el HTML:

npm install -D tailwindcss@3
npx tailwindcss -i tailwind_input.css -o styles.css --minify
⚠️ Las clases generadas dinámicamente en JS (ej: colores, display) NO estarán en el CSS compilado. Usar siempre style.display o estilos inline para visibilidad dinámica.
